# Feature Vector Notes

## Project Title

Global Market AI Investment Prediction System

## Dataset Overview

The dataset contains approximately 120,000 records and 35 columns representing global financial market indicators, sentiment measures, economic conditions, and AI-generated investment recommendations.

The objective is to predict the target variable:

**AI_Investment_Decision**
- Invest
- Monitor
- Avoid

# Categorical Feature Handling

Categorical variables include:

- Market_Type
- Market_Name
- Country
- Asset_Name
- Asset_Sector
- Market_Trend
- Global_Event_Impact

These features are encoded using:

**One-Hot Encoding**

This converts categories into machine-readable binary columns while avoiding artificial ordering.

Example:

Market_Type

- Crypto
- Stock
- Forex
- Commodity

becomes multiple binary features.

# Missing Values

Dataset inspection shows:

| Column | Missing Values |
|----------|----------|
| Global_Event_Impact | ~23,972 |
| Most Other Columns | 0 |

Handling approach:

- Numerical features → Median Imputation
- Categorical features → Most Frequent Value Imputation

This matches the preprocessing pipeline implemented in the project.

# Future Window Leakage Check

The following fields directly influence how the target was created:

- Expected_Return
- Risk_Level
- Technical_Signal
- News_Sentiment

The Streamlit code explicitly removes these columns during training.

Reason:

AI_Investment_Decision is generated using these fields.

Using them during training would allow the model to learn the answer directly rather than learning meaningful patterns.

Status:

Removed from feature vector.

---

# Product Flag Leakage Check

Potential product-generated fields:

- Investment_Recommendation
- Suggested_Investment_Level

These may already contain outputs from another recommendation process.

Using them could leak business logic into the model.

Status:

Should not be used as default predictive features.

---

# Health Score Check

Dataset does not contain:

- health_score

However, similar fields exist:

- Confidence_Score

This field should be reviewed carefully because it may be generated by another model.

Status:

Use cautiously and verify origin before including as a core feature.

---

# Causal Overclaim Check

The model should not claim:

- Market events cause investment success.
- Predictions guarantee future returns.
- AI can accurately forecast all market behavior.

The model only identifies statistical relationships within historical data.

Recommendations should be treated as decision support rather than financial advice.
